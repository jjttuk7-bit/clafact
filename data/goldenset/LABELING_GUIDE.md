# 골든셋 라벨링 가이드 v1.0

> WF-3 (문서 10) 기준 문서. 파일럿 20건 교차 라벨링 후 불일치 논의를 반영해 v2로 갱신한다.
> **이 가이드와 골든셋은 함께 버전 업** — 가이드가 바뀌면 기존 라벨 재검토 대상을 명시할 것.

## 0. 원칙

1. **확신 없으면 판단불가** — 라벨러도 시스템과 같은 원칙을 따른다. 억지 라벨 금지.
2. **근거를 남긴다** — `notes`에 판단 근거(계산식, KOSIS 링크)를 반드시 기록. 근거 없는 라벨은 중재 시 무효.
3. **문장 단위 라벨링** — 판단 단위는 문장(sentence_id). 한 문장에 주장이 2개면 행을 나눈다.
4. **골든셋은 개발 참조 금지 구역** — 라벨링 후 파이프라인 튜닝에 골든셋을 직접 보지 않는다(오염 방지).

## 1. is_claim — 검증 가능 주장인가?

**True 조건 (모두 충족):**
- 수치 또는 수치화 가능한 표현(최상급 포함)이 있다
- 공식 통계(KOSIS)로 참/거짓을 원리적으로 대조할 수 있다
- 사실 진술이다 (의견·평가·인용 발언 아님)

| 문장 | is_claim | 이유 |
|---|---|---|
| 지난해 실업률은 7.2%로 상승했다 | True | 수치 + 공식 통계 대조 가능 |
| 출생아 수는 역대 최저를 기록했다 | True | 수치 없어도 시계열 통계로 검증 가능 (최상급) |
| 경제 상황이 크게 악화되었다 | False | 지표 특정 불가한 정성 서술 |
| 전문가들은 경기 둔화 탓이라고 분석했다 | False | 의견·분석 인용 |
| 장관은 "실업률이 7%대"라고 말했다 | **False (v1)** | 발언 귀속 검증은 범위 밖 (M-6 과제) — v2에서 재논의 |

**경계 사례 규칙:**
- **전망형** ("~할 전망이다", "내년에는 ~예상"): is_claim=True, claim_type=forecast, gold_label=unverifiable
- **복합명사 숫자** ("1인 가구", "3인 가족"): 그 숫자 자체는 주장 아님 (규칙 A2-0006). 문장에 다른 수치가 있으면 그 수치로 판단
- **비유 수치** ("여의도 면적의 3.5배"): 원 수치의 환산 표현 — 원 수치 주장이 별도로 있으면 그쪽만 라벨, 비유만 있으면 True로 하되 notes에 "환산 검증 필요" 기록

## 2. claim_type — 4분류

| 유형 | 코드 | 판단 기준 | 예 |
|---|---|---|---|
| 증감 | increase_decrease | 변화량·변화율이 핵심 ("늘었다/줄었다/상승/하락") | "0.3%p 상승했다" |
| 규모 | scale | 절대 수준이 핵심 ("~이다/달했다/기록했다") | "실업률은 7.2%" |
| 비교 | comparison | 두 대상·시점·집단 간 대비가 핵심 | "OECD 평균의 2배" |
| 전망 | forecast | 미래 시점 진술 — **검증 제외** | "내년 3% 성장 전망" |

한 문장에 증감+규모가 함께 있으면("7.2%로 0.3%p 상승") **행을 2개로 분리**하고 각각 라벨.

## 3. 핵심 요소 추출 (notes에 기록)

- **지표(metric)**: 통계 정식 명칭에 최대한 가깝게 (예: "실업률", "경영주 65세 이상 농가 비율")
- **시점(period)**: 작성일 기준 절대 시점으로 변환 — `YYYY` / `YYYY-MM` / `YYYY-Qn` (규칙 A2-0005). 1월 기사의 "지난달" = 전년 12월 주의
- **단위(unit)**: %, %p 구분 필수. "만 명" 등 스케일 포함 표기 그대로
- **모집단(population)**: 통계의 조사 대상 정의로 (예: "국내 과수 농가" — "농가 전체" 아님)
- **연산자(claimed_op)**: "넘어섰다/돌파" → gte, "밑돌았다/불과" → lte, 그 외 eq

## 4. gold_tbl_id — 정답 통계표 찾기

1. [KOSIS](https://kosis.kr) 검색창에 지표명·조사명 입력 (예: "농림어업조사")
2. 시점·모집단·단위가 주장과 정렬 가능한 통계표를 선택
3. 통계표 URL의 `orgId`, `tblId`를 기록 (예: orgId=101, tblId=DT_1EA1019)
4. **후보가 2개 이상**이면: 주장과 정의가 가장 가까운 것 1개를 gold로, 나머지는 notes에 기록
5. **못 찾으면**: gold_tbl_id 비움 + gold_label=unverifiable + notes에 검색 시도 키워드 기록 (→ 이 기록이 별칭 사전 A1의 원료가 된다)

## 5. gold_label — 판정 3분류

| 라벨 | 기준 |
|---|---|
| match | 시점·단위·모집단 정렬 후 수치 일치 (반올림 고려 — 기사 유효자릿수 기준). 파생 계산(비율·합산)은 **직접 재현 계산**해서 확인하고 계산식을 notes에 기록 |
| mismatch | 정렬 후에도 수치·시점·모집단·표현이 어긋남 → mismatch_type 필수 |
| unverifiable | 지표·시점 불명확 / 대응 통계 부재 / 전망형 |

**mismatch_type**: value(수치) / period(기간) / population(모집단) / exaggeration(과장 표현 — 수치는 근사하나 서술 강도가 통계적 변화를 초과, 예: 1% 감소를 "급감")

## 6. 프로세스 (WF-3)

1. 라벨러 A·B 독립 라벨링 (서로 결과 비공개)
2. 시스템이 일치도(Kappa) 산출 — **0.7 미만이면 중단**, 가이드 재정비 후 재시작
3. 불일치 항목은 제3자 중재로 확정, 반복 패턴은 본 가이드에 규칙으로 추가
4. 버전 태깅 후 발행 (v0.5 → v0.8 → v1.0) — 골든셋 오너([성명]) 승인 필수

## 7. 제출 전 셀프 체크

- [ ] 모든 True 주장에 지표·시점·수치가 notes에 구조화되어 있다
- [ ] match 라벨의 파생 계산은 계산식이 기록되어 있다
- [ ] unverifiable에 사유가 기록되어 있다
- [ ] 시점의 연 경계(1월 기사의 "지난해")를 확인했다
- [ ] %와 %p를 구분했다
