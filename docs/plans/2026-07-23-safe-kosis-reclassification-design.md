# 안전한 KOSIS 재분류 설계

## 목표

기존과 신규 Claim 전체에 새 자동 검증 정책을 적용한다. 직접 KOSIS 표 대조가 가능한 Claim만 자동 검증 큐에 두고, 복잡하거나 근거가 모호한 Claim은 사람 검토 대상으로 분리한다.

## 정책

- `KOSIS_DOMESTIC` 중 직접 수치 대조가 가능한 Claim만 `KOSIS_RETRIEVAL` 및 `PENDING`으로 둔다.
- `KOSIS_BUT_COMPLEX`는 KOSIS 관련이지만 자동 판정하지 않는다. `HUMAN_REVIEW`와 `CLASSIFIED`로 이동한다.
- 기존 `DONE` 결과는 삭제하지 않는다. 재분류 대상이 된 Claim만 결과 필드를 비우고 새 상태로 전환한다.
- 기존 `NON_KOSIS_QUEUE`와 `OUT_OF_SCOPE` 분류는 유지한다.

## 운영 화면

- 검증 탭에서 자동 검증 가능 수, 검토 필요 수, 판단불가 사유별 건수를 보여 준다.
- 운영 홈에서 `전체 새 정책 적용`을 통해 현재 저장된 Claim을 새 분류 정책으로 다시 적용한다.
- 재분류 후 자동 검증 가능 Claim만 일괄 실행한다.

## 안전성

재분류는 멱등이어야 하며, 리뷰가 확정된 Claim은 바꾸지 않는다. 결과를 지우는 경우에도 재분류 사유와 이전 결과를 감사 JSON에 보존한다.

## 검증

- 복잡 KOSIS Claim이 자동 큐가 아닌 검토 경로로 이동하는 테스트
- 직접 KOSIS Claim이 자동 큐에 남는 테스트
- 기존 판정 결과가 감사 정보로 보존되는 테스트
- 화면의 판단불가 사유 집계 표시 테스트
