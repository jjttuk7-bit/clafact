# 운영 홈 가독성·가시성 개선 설계

## 목표

CSV 업로드와 기사 등록 흐름을 운영 홈의 핵심 작업으로 강조하고, 처리 결과와 다음 행동을 명확하게 분리한다.

## 설계

- `운영 실행`을 하나의 작업 카드로 묶고 업로드, 등록 버튼, 진행 상태를 같은 시각적 영역에 둔다.
- 기사 등록 버튼은 넓은 primary 버튼으로 표시해 현재 작업의 주요 행동임을 분명히 한다.
- 진행 상태는 단계형으로 보여준다: 파일 읽기, 기사 등록, 출처 분류, 검증 후보 준비.
- 전처리 수치는 4개 요약 카드로 분리한다: 원본 행, 유효 기사, 수치 주장, KOSIS 분석 대상.
- 분류 결과는 자동 검증 가능, 복합 KOSIS 사람 검토, 별도 근거 확인의 세 가지 색상 카드로 보여준다.
- 하단에는 검증 탭 또는 검증자 리뷰로 이어지는 다음 행동 안내를 둔다.
- 기존 데이터 처리와 집계 로직은 유지하고 Streamlit 표현 계층만 조정한다.
