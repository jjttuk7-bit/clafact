# ClaFact 진단 — PoC의 진짜 상태

작성: 2026-07-20 · 근거: [INVENTORY.md](INVENTORY.md)의 실행 검증 결과만 사용

## 1. 엔드투엔드 관통 여부

**픽스처 세계에서는 관통한다. 실세계에서는 세 지점이 끊겨 있다.**

지금 당장 흐르는 경로 (오늘 실행으로 확인):

```
기사(실물 CSV) → ingest(크롬 제거) → detect(규칙 필터) → parse → retrieve(픽스처 5표)
→ verdict(결정적) → 판정 3분류 + 판단불가 + 재현 URL → 리뷰 큐(서비스층)
```

run_eval 기준 탐지 F1 1.0 / 판정 Acc 1.0 — 단 골든셋 12건 포화 상태라 이 수치는 "관통 증명"이지 "성능 증명"이 아니다.

**끊긴 지점 3개 (정확한 위치):**

| # | 끊긴 곳 | 상태 |
|---|---|---|
| 1 | **detect와 retrieve 사이 — Source Classification 레이어** | `source_classify.py` 미구현, `run.py` 배선 없음. 규칙 v0.1 전수 분류는 일회성 스크립트로만 수행됨(모듈·테스트·시드 파일 없음) |
| 2 | **retrieve의 실세계 확장 — 경로 C 실검색** | 클라이언트는 구현·오프라인 검증됐으나 28만 표 실검색 0회. 개발망이 정부망 443을 차단 → 클라우드 실행 환경 확보가 선결 |
| 3 | **LLM 레이어 — HCX 실호출** | 키는 `.env`에 있으나 실호출 0회. 키 유효성·엔드포인트·쿼터 전부 미검증. `detect_llm.py`는 파일도 없음 |

## 2. G-SOURCE-1 게이트 통과 가능성

**지금 상태로는 불통과. 결핍은 4개이며 전부 하루 단위 작업이다.**

게이트 요구(기획 문서) vs 현재:

| 완료 기준 | 현재 |
|---|---|
| ① source_type 분포 생성 | ✅ 사실상 완료 — 전수 16,464건 분포 존재(공유자료 19). 단 50건 파일럿의 사람 검증 라벨은 없음 |
| ② KOSIS/비-KOSIS 분리 | ✅ 분류 결과로 분리 가능 (KOSIS 후보 820건) |
| ③ UNKNOWN 사람 검토 큐 | ❌ 큐 없음 — UNKNOWN 10,846건이 파일에만 있음 |
| ④ 오매핑 사례 목록 | 🔶 시드 3건 발굴(공유자료 19 §6) — 목록 파일 없음 |
| ⑤ routing rule v0.1 | ❌ `routing_v01.json` 파일 없음 (규칙이 스크립트 코드 안에만 존재) |

**결론: 분석은 게이트를 앞질러 갔는데 산출물 형식이 뒤처져 있다.** 스크립트 로직을 `source_classify.py` + `routing_v01.json` + 테스트로 승격시키고, 파일럿 50건에 사람 라벨을 붙이면 통과한다.

## 3. 골든셋 상태

- 실물: `golden_v0.jsonl` **12건** (탐지·판정용 시드, 사람 라벨) + `mapping_eval.jsonl` 10건 (매핑용).
- **"20건 파일럿(경제 8, 사회 8, 파생 2, 판단불가 2)"은 존재하지 않는다** — 계획 문서 속 목표가 실물로 오인된 사례.
- 12건 시드는 사람 라벨이 맞고 오염 없음(릴리스 게이트의 골든셋 오염 검사 PASS).
- 파일럿 50건의 원료는 준비됨: KOSIS 후보 820건 + 층화 추출안(인구가구20/고용15/물가10/농가5). **부족한 것은 코드가 아니라 사람의 라벨링 시간이다.**

## 4. 5축 커버리지 — 검증 수준이 축마다 다르다

| 축 | 픽스처 E2E | 실 KOSIS API 검증 |
|---|---|---|
| 농가 고령화 (과수) | ✅ (DT_1EA1019) | ✅ 2026-07-14 실 API 대조 기록 |
| 고용·실업률 | ✅ | ❌ |
| 인구·출생아 | ✅ | ❌ |
| 1인 가구 | ✅ | ❌ |
| CPI (물가) | ✅ (DT_1J22112, 기준연도 규칙 포함) | ❌ |

즉 **"실데이터 검증 완료"라고 말할 수 있는 축은 1개뿐**이다. 나머지 4축의 실 API 검증은 운영계정·클라우드 확보에 걸려 있다.

## 5. 리스크 상위 3 — 가장 먼저 죽는 지점

1. **경로 C 실측 지연 (가장 먼저 죽는다)**: W1 게이트 G1이 클라우드 실행 환경이라는 외부 조건에 걸려 있다. 환경 확보가 하루 늦어질 때마다 G1이 하루 밀리고, G1이 밀리면 매핑 경로 결정 → EXP-001(W3) → 대량 처리(W4)가 연쇄로 민다. **오늘 클라우드에서 `search_kosis()` 1회 실행이 최우선 방어선이다.**
2. **골든셋 파일럿의 인력 의존**: 라벨링은 자동화 금지(가드레일)라 사람 시간이 유일한 자원인데, 휴가 전 실작업일이 12일뿐이다. 표본 100건 읽기와 라벨링이 W1 후반으로 밀리면 휴가가 흐름을 끊는다.
3. **HCX 미검증 상태로 W2 진입**: 키 유효성·요청 포맷·쿼터를 모른 채 W2를 시작하면, "연동"이 아니라 "디버깅"으로 한 주를 쓴다. 오늘 1회 스모크 호출로 이 리스크는 10분에 제거된다.
