> Author: Human Team + ClaFact Hermes Agent
> Reviewed by: Human Team
> Managed by: ClaFact Hermes Agent
> Status: Draft
> Version: v0.1
> Last Updated: 2026-07-22

## 사용 방법

- 각 역할 담당자는 자신의 번호가 붙은 항목을 Daily Log와 Backlog에서 참조한다.
- `[ ]`는 미완료, `[x]`는 완료를 의미한다.
- `연결 역할`은 입력을 받거나 결과를 전달해야 하는 역할이다.
- 확정된 정책·결정은 Daily Log가 아니라 `ops/DECISION_LOG.md`에 승격한다.
- Discord는 논의 공간이며, 충돌 시 GitHub 공식 문서를 우선한다.

## 전체 운영 흐름

`R2 Claim 추출` → `R3 Evidence 매핑` → `R4 Verdict·Service 판정` → `R1 Evaluation 평가·릴리스 게이트` → `R5 Hermes 브리핑·자동화` → 실패·결정 내용을 각 역할에 재배분

## R1. PM·Evaluation

책임: 프로젝트의 품질·진척·릴리스 가능 여부를 숫자와 근거로 판단한다.

### R1-001 평가 목표와 범위 확정

- [ ] 이번 평가의 대상 버전, 데이터셋, Claim 범위, 제외 범위를 확정한다.
- 산출물: 평가 계획서, 대상 커밋·데이터 버전
- 연결 역할: R2(Claim 범위), R3(Evidence 범위), R4(Verdict 기준)

### R1-002 골든셋 구성 계획 수립

- [ ] 사람 라벨 골든셋을 최소 200~500건 규모로 구성하고 클래스·분야·난이도 분포를 정의한다.
- 산출물: 골든셋 구성표, 라벨링 계획
- 연결 역할: R2(Claim 라벨), R3(통계표 매핑 라벨), R4(Verdict 라벨)

### R1-003 라벨링 가이드 승인

- [ ] Claim 여부, 통계표 ID, 판정, 판단불가 사유, 오류 유형의 라벨 정의를 확정한다.
- 산출물: 라벨링 가이드, 예시·경계 사례 목록
- 연결 역할: R2, R3, R4

### R1-004 라벨 품질 관리

- [ ] 2인 교차 라벨링, 불일치율, 합의 규칙, 중재자를 기록한다.
- 산출물: 라벨 품질 리포트, 불일치 목록
- 연결 역할: R2(Claim 중재), R3(Evidence 중재), R4(Verdict 중재)

### R1-005 평가 데이터 분할

- [ ] 학습·개발·테스트 데이터가 누수되지 않도록 분할하고, 동일 기사·동일 통계표의 중복 여부를 점검한다.
- 산출물: 데이터 분할 manifest, 누수 점검 결과
- 연결 역할: R2, R3, R5(재현 실행)

### R1-006 기준선 실험 설계

- [ ] 키워드·BM25, 임베딩 검색, LLM 단독, RAG 방식과 ClaFact를 동일 조건으로 비교한다.
- 산출물: baseline 실행 명세, 비교표 템플릿
- 연결 역할: R3(검색 baseline), R4(판정 baseline), R5(실행 자동화)

### R1-007 핵심 평가 지표 확정

- [ ] Claim 탐지 F1, 매핑 Hit@k·MRR, Verdict Accuracy·Macro-F1, 판단불가 정확도, 인용 정확도를 정의한다.
- 산출물: 지표 정의서, 계산식·성공 기준
- 연결 역할: R2(탐지), R3(매핑), R4(판정·판단불가)

### R1-008 릴리스 게이트 정의

- [ ] 정확도만 보지 않고 오판율, 판단불가 비중, 설명 오류, 테스트·보안 검사 결과를 함께 평가한다.
- 산출물: 릴리스 게이트 기준, PASS·FAIL 규칙
- 연결 역할: R4(발행 기준), R5(자동 게이트 실행)

### R1-009 Ablation 실험 계획

- [ ] 규칙 카드, 리랭커, 단위 정규화, 모집단 정렬, 판단불가 정책을 제거하는 실험을 설계한다.
- 산출물: Ablation matrix, 실험 우선순위
- 연결 역할: R2(규칙 카드), R3(리랭커·매핑), R4(판정 정책)

### R1-010 주간 평가 실행

- [ ] 최신 커밋과 고정된 데이터 버전으로 평가를 실행하고 전 회차 대비 변화를 기록한다.
- 산출물: 평가 리포트, 전 회차 diff
- 연결 역할: R2, R3, R4(실패 원인 설명), R5(실행·브리핑)

### R1-011 오류 유형별 분석

- [ ] 탐지·매핑·시점·단위·모집단·판정·설명 오류를 분류하고 우선순위를 정한다.
- 산출물: 오류 분석표, 다음 Sprint 우선순위
- 연결 역할: R2(Claim 오류), R3(Evidence 오류), R4(Verdict 오류)

### R1-012 실제 데이터 사례 검토

- [ ] 실제 기사·공식 통계·계산식·최종 판정이 연결된 사례를 표본 검토한다.
- 산출물: 사례 검토표, 성공·실패 대표 사례
- 연결 역할: R3(근거), R4(판정), R5(재현 링크)

### R1-013 릴리스 게이트 판정

- [ ] 대상 커밋, 평가 결과, 실패 임계값, 승인자, 보류 사유를 기록해 PASS·HOLD·FAIL을 판정한다.
- 산출물: 릴리스 게이트 판정, `ops/DECISION_LOG.md` 기록 요청
- 연결 역할: R4(서비스 발행 보류), R5(배포·브리핑)

### R1-014 금요일 체크아웃 작성

- [ ] 주간 진행, 숫자 변화, 주요 실패, 확정 결정, 다음 주 우선순위를 통합한다.
- 산출물: 주간 체크아웃 문서
- 연결 역할: R2, R3, R4, R5(각 Daily·Backlog)

### R1-015 연구·발표용 증거 패키지 관리

- [ ] 데이터셋, 실행 명령, 커밋, 결과 리포트, 오류 분석, 한계를 발표용 패키지로 묶는다.
- 산출물: 논문용 실험 부록, 재현 패키지
- 연결 역할: R5(재현 자동화), 전 역할(근거·실패 사례 제공)

## R2. Claim

책임: 뉴스와 콘텐츠에서 검증 가능한 수치 Claim을 추출·정규화·분류한다.

### R2-001 Claim 추출 범위 확정

- [ ] 사실 여부를 공식 통계로 확인할 수 있는 문장과 제외할 문장을 구분한다.
- 산출물: Claim 추출 기준
- 연결 역할: R1(평가 범위), R3(매핑 가능성)

### R2-002 원문 수집·식별

- [ ] 기사 URL·본문·발행일·출처를 저장하고 동일 기사의 중복을 제거한다.
- 산출물: 원문 manifest
- 연결 역할: R3(검색 입력), R5(수집 자동화)

### R2-003 Claim 후보 추출

- [ ] 수치·비율·증감·순위·비교 표현을 후보 Claim으로 추출한다.
- 산출물: Claim 후보 JSONL 또는 표
- 연결 역할: R1(탐지 평가), R3(매핑 입력)

### R2-004 Claim 구조화

- [ ] 주체, 지표, 값, 단위, 기간, 비교 기준, 모집단, 방향을 필드로 분리한다.
- 산출물: 정규화 Claim 카드
- 연결 역할: R3(검색 쿼리), R4(판정 입력)

### R2-005 제외 규칙 적용

- [ ] 의견·감정·예측·검증 불가능한 표현을 제외하거나 별도 표시한다.
- 산출물: 제외 목록과 사유
- 연결 역할: R1(탐지 평가), R4(판단불가 분석)

### R2-006 오류 유형 기록

- [ ] 단위 누락, 기간 누락, 비교 기준 누락, 주체 불명확을 분류한다.
- 산출물: Claim 오류 태그
- 연결 역할: R1(오류 분석), R3(매핑 보정), R4(판정 보류)

### R2-007 Claim 라벨링

- [ ] 사람 라벨러가 Claim 여부와 오류 유형을 라벨링하고 불일치를 중재한다.
- 산출물: 사람 라벨 골든셋
- 연결 역할: R1(라벨 품질), R5(라벨 파일 관리)

### R2-008 Claim 변경 추적

- [ ] 추출 규칙·프롬프트·파서 변경 전후의 Claim 수와 샘플 변화를 기록한다.
- 산출물: 변경 diff, 회귀 사례
- 연결 역할: R1(회귀 평가), R5(자동 실행)

### R2-009 매핑 요청 전달

- [ ] R3에 Claim 카드, 검색 핵심어, 기간·단위·주체, 우선순위를 전달한다.
- 산출물: Evidence 매핑 큐
- 연결 역할: R3(필수), R4(판정 사전 확인)

### R2-010 매핑 결과 검수

- [ ] R3가 반환한 통계표·지표·단위가 원문 Claim과 의미적으로 일치하는지 검수한다.
- 산출물: 매핑 승인·반려·보류 기록
- 연결 역할: R3(근거), R4(판정)

### R2-011 실패 사례 개선

- [ ] 반복 실패를 규칙 카드, 별칭 사전, 추출 예외로 승격할 후보를 만든다.
- 산출물: 개선 후보, 회귀 테스트 요청
- 연결 역할: R1(우선순위), R3(검색 별칭), R5(자동화 반영)

### R2-012 Daily·Backlog 갱신

- [ ] 오늘 처리량, 대표 실패, 막힌 점, R3·R4 요청, 내일 작업을 기록한다.
- 산출물: R2 Daily Log, Backlog 상태
- 연결 역할: R1(주간 평가), R5(오전 브리핑)

## R3. Evidence

책임: Claim을 검증할 수 있는 공식 Evidence를 검색·매핑·검수한다.

### R3-001 공식 출처 범위 확정

- [ ] KOSIS 통계표·기관·API·조회 시점 등 허용 출처와 우선순위를 정의한다.
- 산출물: 출처 정책
- 연결 역할: R1(평가 기준), R4(발행 근거 기준)

### R3-002 검색 입력 정규화

- [ ] R2 Claim의 주체·지표·기간·단위·비교 기준을 검색 필드로 변환한다.
- 산출물: 검색 쿼리·정규화 로그
- 연결 역할: R2(Claim 입력), R5(검색 자동화)

### R3-003 후보 통계표 검색

- [ ] 키워드·BM25·임베딩 등 후보 검색 경로를 실행하고 상위 결과를 저장한다.
- 산출물: 후보 목록, 검색 점수
- 연결 역할: R1(baseline), R4(판정 입력)

### R3-004 리랭킹 적용

- [ ] 통계표명, 지표 정의, 단위, 주기, 분류, 주석을 사용해 후보 순위를 재정렬한다.
- 산출물: 리랭킹 결과와 선택 사유
- 연결 역할: R1(Ablation), R2(Claim 의미 검수)

### R3-005 통계표 매핑 확정

- [ ] 통계표 ID, 지표, 분류값, 단위, 기간, 조회일을 기록한다.
- 산출물: Evidence 매핑 카드
- 연결 역할: R2(매핑 승인), R4(판정)

### R3-006 원자료 조회·보존

- [ ] 실제 API 응답 또는 재현 가능한 조회 정보를 버전과 함께 보존한다.
- 산출물: 원자료 snapshot, 재현 URL
- 연결 역할: R4(근거 사용), R5(재현 자동화)

### R3-007 계산용 값 정리

- [ ] 비교에 필요한 원자료 값, 단위, 기간, 모집단, 기준값을 추출한다.
- 산출물: 계산 입력 레코드
- 연결 역할: R4(결정적 판정), R1(계산 재현성 평가)

### R3-008 Evidence 품질 검수

- [ ] Claim의 주체·지표·기간·단위·모집단과 Evidence 정의가 일치하는지 확인한다.
- 산출물: Evidence 승인·반려·판단불가 사유
- 연결 역할: R2(의미 검수), R4(판정 정책)

### R3-009 매핑 평가셋 구축

- [ ] 사람이 정답 통계표를 라벨링한 매핑 평가셋을 구성하고 Hit@k·MRR을 측정한다.
- 산출물: 매핑 골든셋·평가 리포트
- 연결 역할: R1(성능 평가), R2(Claim 샘플), R5(실행)

### R3-010 실패 유형 분석

- [ ] 검색어 부족, 별칭 불일치, 통계표 정의 불일치, 기간·단위 오류를 분류한다.
- 산출물: Evidence 오류 분석표
- 연결 역할: R1(우선순위), R2(검색어 개선), R4(판정 보류)

### R3-011 Verdict 전달

- [ ] R4에 승인된 Evidence, 계산 입력, 출처, 조회 시점, 불확실성을 전달한다.
- 산출물: Verdict 입력 큐
- 연결 역할: R4(필수), R1(사례 평가)

### R3-012 Daily·Backlog 갱신

- [ ] 검색량, 매핑 성공·실패, 주요 근거, R2·R4 요청, 다음 작업을 기록한다.
- 산출물: R3 Daily Log, Backlog 상태
- 연결 역할: R1(주간 평가), R5(오전 브리핑)

## R4. Verdict·Service

책임: 승인된 Evidence에 근거해 Verdict를 계산·설명하고 사람 리뷰·발행 흐름을 관리한다.

### R4-001 Verdict 라벨 정의

- [ ] 일치, 불일치, 판단불가의 정의와 경계 사례를 확정한다.
- 산출물: Verdict 정책
- 연결 역할: R1(평가 지표), R2(Claim 의미), R3(Evidence 품질)

### R4-002 판단불가 사유 체계화

- [ ] 기간·단위·모집단 불일치, 근거 부족, 주체 불명확, 출처 미확인 사유를 코드화한다.
- 산출물: 판단불가 사유 코드
- 연결 역할: R1(판단불가 평가), R2·R3(부족한 입력)

### R4-003 발행등급 기준 확정

- [ ] 근거 링크, 조회 시점, 계산식, 판정 사유, 리뷰 상태를 충족해야 발행하도록 정한다.
- 산출물: 발행등급 기준
- 연결 역할: R1(릴리스 게이트), R5(자동 게이트)

### R4-004 결정적 판정 실행

- [ ] LLM과 분리된 코드로 단위 환산, 반올림, 비교, 파생 계산을 실행한다.

## 역할 간 인계 요약

| 인계 | 전달 내용 | 완료 조건 |
|---|---|---|
| R2 → R3 | 구조화 Claim, 검색어, 기간·단위·주체 | 매핑 가능한 Claim 카드가 있음 |
| R3 → R4 | 통계표 ID, 원자료 값, 단위, 기간, 재현 URL | 계산 가능한 Evidence 카드가 있음 |
| R4 → R1 | Verdict, 판단불가 사유, 리뷰 결과, 오류 유형 | 평가 가능한 결과와 근거가 있음 |
