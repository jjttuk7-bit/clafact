역할 간 인계 요약

인계전달 내용완료 조건
R2 → R3구조화 Claim, 검색어, 기간·단위·주체매핑 가능한 Claim 카드
R3 → R4통계표 ID, 원자료 값, 단위, 기간, 재현 URL계산 가능한 Evidence 카드
R4 → R1Verdict, 판단불가 사유, 리뷰 결과, 오류 유형평가 가능한 결과와 근거
R1 → R5평가 기준, 릴리스 게이트, 우선순위자동 실행 가능한 기준
R5 → 전 역할브리핑, 문서 상태, 실패·블로커, 다음 요청각 역할이 다음 작업을 인지

주기별 운영 체크

  • 매일: R2~R5 Daily Log 갱신 → R5 오전 브리핑 → 역할 요청을 Backlog에 반영
  • 매주: R1 평가·체크아웃 → R5 수집 → 완료 항목을 Completed로 이동
  • 릴리스 전: R1 게이트 판정 → R4 발행·리뷰 확인 → R5 테스트·보안·재현성 검사
